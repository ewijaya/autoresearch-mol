\begin{abstract}
Molecular transformers typically borrow architectures from natural language processing without systematic evaluation of domain fit. We ask: do molecular sequences require fundamentally different transformer designs, and does autonomous architecture search add value beyond hyperparameter tuning? To answer this, we deploy an LLM-guided architecture search agent across three sequence domains---SMILES (small molecules), protein sequences, and English text---using a 4-condition experimental design that decomposes the contributions of architecture search versus hyperparameter (HP) tuning. Across 3{,}106 experiments on a single A10G GPU, we find that the value of architecture search is \emph{domain-dependent}: for NLP (large vocabulary, long sequences), architecture search contributes 81\% of total improvement ($p = 0.008$), while for SMILES (small vocabulary, short sequences), HP tuning alone captures 151\% of the improvement and architecture search is counterproductive ($p = 0.001$). Despite this divergence, agent-discovered architectures cluster by domain ($p = 0.004$), yet all 41 discovered innovations transfer across domains with $<$1\% degradation---suggesting that search dynamics, not fundamental domain requirements, drive architectural specialization at this scale. We provide a practitioner decision framework linking domain characteristics (vocabulary size, sequence length) to the recommended search strategy, and release an open-source framework for autonomous architecture search on arbitrary sequence domains.
\end{abstract}
